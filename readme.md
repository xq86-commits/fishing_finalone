# 🎮 项目结构说明

本项目采用「**单页框架 + 独立游戏页 iframe 嵌入**」的架构。
主框架 `index.html` 管理启动页、排行榜、结算与加载逻辑；
具体游戏逻辑由 `game.html` + `game.js` 组成。

---

## 🏠 **index.html**

**作用：**
作为游戏网站的主页面（入口页 + 排行榜页 + 游戏容器）。
所有 UI 与页面切换逻辑都在这里完成。

**主要模块：**

* 🎯 **首页（Home）**：显示游戏标题、开始按钮、排行榜入口。
* 🕹️ **游戏 iframe**：嵌入 `/games/game.html`，通过 `postMessage` 与父页面通信。
* 🏆 **排行榜页（Leaderboard）**：显示示例排名列表，支持返回首页。
* ⏳ **加载动画（Loading）**：在切换到游戏前显示短暂 loading。

**核心逻辑：**

* 点击「开始游戏」 → 隐藏首页 → 显示加载动画 → 进入游戏页（iframe）
* 点击「排行榜」 → 展示排行榜列表
* 游戏结束后（来自 iframe 的 `levelComplete` 消息） → 弹出提示 + 返回首页

**适配特点：**

* 使用 `--scale: calc(100vw / 375)` 动态缩放
* iOS / Android 全端自适应
* 禁止双击缩放、文字选中、长按菜单

---

## 🎮 **game.html**

**作用：**
游戏实际运行页面，独立在 iframe 中执行。
负责 Canvas 渲染与触控事件，不依赖外部样式文件。

**主要内容：**

* `<canvas id="game">`：主画布
* 内嵌 CSS：控制背景、比例缩放、圆角与阴影
* 防缩放与防长按逻辑（CSS + JS）
* 加载 `game.js`，执行具体玩法

**移动端优化：**

* ✅ 禁止双击放大（`touch-action: manipulation` + JS 防抖）
* ✅ 禁止长按复制、保存图片
* ✅ 高清屏幕渲染（支持 DPR=2/3）
* ✅ Canvas 尺寸自动根据 `--scale` 缩放

---

## 🧩 **game.js**

**作用：**
核心游戏逻辑脚本（示例：点击气球小游戏）。
负责游戏初始化、绘制、动画循环、得分统计与父页面通信。

**主要流程：**

1. **setupCanvas()**：根据 `devicePixelRatio` 调整 Canvas 高清绘制
2. **spawn()**：生成随机气球对象（颜色、速度、位置）
3. **loop()**：每帧清屏 → 绘制气球 → 检测结束条件
4. **点击事件**：检测命中气球，增加分数并移除目标
5. **关卡完成**：当 `score >= 20` 时发送消息给父页面：

   ```js
   window.parent.postMessage({ action: "levelComplete", score }, "*");
   ```

   由 `index.html` 接收处理。

**特色：**

* 高分屏适配：根据 DPR 动态缩放
* 轻量动画循环：`requestAnimationFrame`
* 简单交互逻辑，可替换为其他玩法

## 📁 文件关系结构

```plaintext
project-root/
│
├── index.html         # 主页面（首页 + 排行榜 + iframe 容器）
│
└── /games/
    ├── game.html      # 游戏页面（Canvas + 样式 + 触控优化）
    └── game.js        # 游戏逻辑脚本（绘制与通信）
```


---

> ✅ **一句话总结：**
> `index.html` 管控整体结构与导航，`game.html` 专注展示游戏画面，`game.js` 负责具体玩法逻辑。
> 三者独立又协作，结构简洁、扩展性强，非常适合做「多游戏合集」或「小游戏导航站」的基础框架。
